# 更新日志

## v1.0.5 - 2026-07-23

### 新增

- 新增 `OpenFileLogger`，显式返回动态 logger 创建错误。
- 新增 `Flush` 和 `Shutdown`，允许调用方处理同步与关闭错误。
- 新增 `ErrNotInitialized`，标识动态 logger 调用不在有效生命周期内。
- 新增 Windows 和 Linux 上的 Go 1.24.0、Go 1.26.4 CI 验证。

### 修复

- 消除生命周期竞态并安全停止每日轮转调度器。
- 让初始化后创建的动态滚动 logger 加入午夜轮转。
- 让指向同一物理路径的动态 logger 复用同一个 writer。
- 拒绝路径穿越、配置路径冲突和调用时已存在的越界符号链接。
- 恢复准确的 caller 信息和标准库 logger 状态。
- 清理跨 generation 缓存并聚合受管理资源错误。
- 在 `Close` 或 `Shutdown` 后保持动态 logger 不可用。
- 在保留文件同步的同时避免控制台产生不受支持的 `fsync` 错误。
- 让 `Flush` 和 `Shutdown` 同步 rolling writer 当前活动文件。
- 让午夜轮转在夏令时切换日仍按本地日历的下一次 00:00 触发。

### 兼容性

- 保留现有 `FileLogger`、`Sync` 和 `Close` 签名。
- 过去可能被接受的非法路径配置现在会返回错误。
- 重复调用 `Init` 可能返回上一代 logger 的清理错误；发生这种情况时，新配置已经生效。

### 安全边界

- 路径校验防止不可信相对路径输入和校验时可见的符号链接越界。
- 配置的日志根目录必须归可信主体所有，并避免不可信本机进程并发替换目录结构。
